## To the Reviewer

Thank you for reviewing Release Pilot.

Release Pilot is an App Store Connect companion built for solo iOS developers. It connects to your own App Store Connect account using an **App Store Connect API key that you (the developer) generate yourself in App Store Connect → Users and Access → Integrations → Keys**. The app then reads release/build/review data from Apple's public App Store Connect API directly from the device, and surfaces it in a single multi-app dashboard, push notifications, widgets, and a Live Activity.

### How the App Store Connect API key is handled

This is the most security-sensitive part of the app. We want to be transparent:

1. **The key is generated by the user, in App Store Connect, under their own Apple Developer account.** The app provides step-by-step instructions on how to do this (see the screenshot of the "Get your API key" onboarding screen, attached as Screenshot 1).

2. **The key never leaves the device.** When the user pastes the .p8 file contents, the Key ID, and the Issuer ID into the app, they are written immediately to the iOS Keychain via `expo-secure-store` with `kSecAttrAccessibleWhenUnlockedThisDeviceOnly` and biometric protection (Face ID / Touch ID). No copy is ever made to local storage, no copy is ever sent over the network to any server.

3. **JWT generation happens on-device.** Every request to `https://api.appstoreconnect.apple.com/...` is signed locally using the ES256 algorithm against the user's own private key. The JWT lives in memory for at most 19 minutes (per Apple's recommendation), then is regenerated.

4. **We do not operate any server that proxies, caches, mirrors, or logs App Store Connect data.** The only server-side component of this app is a Cloudflare Worker that delivers push notifications. The Worker receives **only** an APNs device token and a state-change payload (e.g. `{app_id, from_state, to_state}`). It never sees the user's API key or any other ASC credentials.

5. **Revocation is instant and user-controlled.** Users can delete the key from Release Pilot from More → ASC Accounts at any time, and they can revoke the key in App Store Connect at any time, which immediately renders Release Pilot inert.

The app uses the App Store Connect API for exactly these operations, all initiated by explicit user action:
- `GET /v1/apps` — list the user's apps (refreshed on pull-to-refresh and background fetch)
- `GET /v1/apps/{id}/appStoreVersions` — version history
- `GET /v1/customerReviews` — reviews for replying
- `POST /v1/customerReviewResponses` — submit a review reply (only after the user taps "Reply")
- `GET /v1/builds` — TestFlight builds for the pre-submit checklist

The app does **not** submit apps for review, change pricing, or take any irreversible action on the user's account.

### Account deletion (Guideline 5.1.1(v))

Release Pilot has no traditional user account — credentials live on-device in iOS Keychain — but we still provide a one-tap "delete account" equivalent for full guideline compliance:

**Single destructive action:** **More → DANGER ZONE → Erase all data**

That screen wipes, in one operation:

1. Every connected App Store Connect API key (.p8 PEMs in iOS Keychain)
2. Every RevenueCat secret key (per-app, in iOS Keychain)
3. All cached app data (SQLite tables: app rows, version history, customer reviews, offline reply queue)
4. All MMKV-backed app state (gate counters, briefing snapshot, Live Activity records)
5. The push registration row on our worker (`POST /v1/unregister`) — Apple stops being polled on behalf of this device
6. The RevenueCat subscriber alias (`Purchases.logOut()`)
7. All scheduled iOS notifications
8. The shared App Group container the widget reads from (Home / Lock Screen widgets revert to empty state)
9. Any active Live Activities

The user is then routed back to the welcome / onboarding screen. The app's state is indistinguishable from a fresh install at that point.

We also retain the granular delete affordances — More → ASC Accounts → swipe-to-delete, and More → Apps → disconnect RevenueCat — for users who want to remove a single team or RC project without nuking the whole device.

**What the wipe does NOT do** (and we explain this in-line on the screen):

- It does not cancel the App Store subscription itself — Apple owns that and the user manages it via Settings → Apple ID → Subscriptions.
- It does not revoke the API key in App Store Connect — only Apple's own ASC website can do that.

### Background refresh

Release Pilot uses iOS Background Refresh (`UIBackgroundModes: ['fetch', 'processing', 'remote-notification']`) to poll the App Store Connect API every ~30 minutes when the app is backgrounded. This is what drives push notifications without us needing a server-side polling job. The poll respects iOS's adaptive scheduling and silently no-ops on a metered network.

### Push notifications

Push notifications are delivered via APNs through a thin Cloudflare Worker at `release-pilot.workers.dev`. The Worker is a stateless proxy that takes a device token + a payload and signs the APNs request. It receives the payload from the **client** (the user's own iPhone, not from any third party). The user must explicitly grant notification permission, which is requested only after the user has connected an ASC account and tapped the "Enable notifications" toggle.

### Live Activities

Live Activities are started automatically when an app the user has connected enters the "Waiting for Review" state, and ended when it transitions to "In Review" or "Rejected." This is informational only — the Live Activity is read-only.

### Privacy

- We do not collect personally identifiable information from our users (no analytics, no telemetry, no third-party SDKs aside from RevenueCat for IAP receipt validation)
- RevenueCat (used to validate IAP receipts and unlock Pro features) receives an anonymous installation ID generated by the user's device. It receives no Apple ID, no name, no email, and no ASC data.
