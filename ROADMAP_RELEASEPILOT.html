V1 (MVP) — Remaining before launch + post-submission queue 3 items

v1.0 binary is submitted and under Apple review. These are the only items still between “awaiting verdict” and “earning money.” The v1.0.1 backlog row tracks JS/TS + native changes already committed to main that will ship in the next EAS build.

Feature Area Status Notes
v1.0 Apple App Review Release In progress Production build uploaded via eas submit; binary in Apple's review queue. Typical turnaround 24–48h. No OTA updates pushed during review — v1.0.1 backlog stays parked on main until approval.
Briefing widget (glanceable Home + Lock variant) Native Partial In-app Today tab + 7am local push shipped. Standalone glanceable widget is descoped pending user demand — the in-app + push surfaces deliver the value today; widget adds a third surface for the same data and isn't a launch blocker. Would require another EAS build when prioritized.
v1.0.1 post-approval backlog (parked on main) Release In progress Commits on main waiting on v1.0 approval before shipping (either bundled into a v1.0.1 EAS build or OTA'd):
  • Checklist 15 → 17 rules — catches the missing-price-tier + missing-territory rejection (OTA-able).
  • Sticky daily 'State changes' counter — baseline anchored at 7am local so multi-app changes accumulate through the day instead of resetting on tab re-open (OTA-able).
  • Today tab hero subtitle — "Since 7:00 AM · resets each morning" caption so the 'State changes' metric is unambiguous (OTA-able).
  • Paywall Terms link URL — points at the custom hosted Terms instead of Apple's generic stdeula, matching the App Store Description (OTA-able).
  • Lock-Screen circular widget label — LIVE → WIP with a checkmark when count is 0 (native, requires EAS build).
  • Planning docs + Terms hosting — progressive app picker design, App Preview production plan, docs/terms.html on GitHub Pages (not bundled, repo / static-site only).
